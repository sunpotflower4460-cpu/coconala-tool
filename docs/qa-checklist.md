# QA Checklist

このチェックリストを使ってアプリの品質を確認してください。

## セットアップ確認

- [ ] `npm ci` が完了する（lockfile から再現インストール、エラーなし）
- [ ] `npm run lint`（型チェック）が通る（TypeScript エラーなし）
- [ ] `npm run build` が通る（`dist/` が生成される）
- [ ] `npm run test` が通る（利益計算 / CSV出力テスト）
- [ ] `npm run dev` でローカル起動し、ブラウザで表示される

## デモ表示・データソース

- [ ] ヘッダーに「デモ表示中 — サンプル/モックデータ」バッジが常時表示され、スクロールしても消えない
- [ ] サンプルカードに `サンプルデータ`、モックカードに `モック（楽天API想定）` バッジが表示される
- [ ] データソース切替（サンプル / 楽天市場）に各モードの一行説明が表示される
- [ ] 検索中はボタンが「検索中…」になり無効化され、二重送信できない
- [ ] 楽天市場モードでキー未設定・タイムアウト・通信失敗・レート超過・上流エラー時に、理由が検索結果の上に表示される
- [ ] 楽天API接続で実データが取得できた場合、ヘッダーが緑色の「公式データ取得中」表示に切り替わる
- [ ] API接続準備ステータスに「現在のデータソース」が表示される
- [ ] 実価格と誤認させる断定表現がどの画面にもない

## 検索フロー

- [ ] 商品名・型番を入力して「まとめて探す」を押すと結果カードが表示される
- [ ] 検索ショートカットカード（メルカリ・ヤフオク等のリンク）が表示される
- [ ] 「元ページを見る」リンクが外部ページに正しく開く

## 比較フロー

- [ ] カードの「比較に追加」を押すと、右サイドの比較ボードに追加される
- [ ] 比較ボードの「×」でカードを削除できる
- [ ] 比較ボードが空のとき、空状態メッセージが表示される
- [ ] 「この価格を仕入れに使う」「この価格を販売に使う」で利益パネルの数値が変わる

## 利益クイック入力フロー

- [ ] 仕入れ・販売・送料・手数料・為替レートを入力すると利益が計算される
- [ ] USD カードがある場合、円換算が表示される
- [ ] 利益がマイナスや 10% 未満のとき AI メモに警告が出る

## 手動追加フロー

- [ ] 「手動で追加」ボタンからモーダルが開く
- [ ] 無効なURL（例: `javascript:void`）を入力するとエラーメッセージが出る
- [ ] 価格未入力の場合「価格不明」と表示されることを確認できる（ヒント表示される）
- [ ] 同一URLを入力すると重複警告メッセージが表示される
- [ ] 正常な URL・価格を入力して「比較に追加」を押すとカードが追加される

## CSV エクスポート

- [ ] 比較ボードにカードを追加した状態で「比較中カードをCSV出力」を押すとファイルがダウンロードされる
- [ ] 比較ボードが空の場合、CSV ボタンが無効になり空状態メッセージが表示される
- [ ] ダウンロードされた CSV を Excel/スプレッドシートで開いて文字化けしないことを確認

## 履歴保存・再開・削除

- [ ] 「保存」ボタンで現在のリサーチが履歴に保存される
- [ ] 保存名を入力して保存すると、その名前で表示される
- [ ] 「再開」で保存時の状態が復元される
- [ ] 「削除」で履歴から項目が消える
- [ ] 履歴が空のとき、空状態メッセージが表示される

## テーマ切替

- [ ] ヘッダー右上のテーマセレクタで全テーマに切り替えられる
- [ ] テーマがブラウザ再読み込み後も維持される

## ソースラベル確認

- [ ] カードに `公式API取得` / `検索表示から推定` / `検索リンク` / `手動追加` ラベルが表示される
- [ ] 検索リンク系カードに「外部検索ページ」バッジが表示される

## ポリシー確認

- [ ] フロントエンドコードにAPIキー・シークレット等の固定値がない
- [ ] スクレイピング・高頻度アクセスコードが含まれていない
- [ ] 価格が「推定」として明示されている（断定文がない）

## モバイル表示確認

- [ ] 幅 375px（iPhone SE相当）でレイアウトが崩れない
- [ ] 右サイドパネルが縦積みになる
- [ ] 長い商品名・URLが折り返し/省略されてレイアウトが崩れない

## アクセシビリティ確認（グラスUI）

- [ ] すべての操作要素で Tab キーのフォーカスが見える（アクセント色の枠線）／ Tab 順が自然
- [ ] 削除「×」・閉じるボタンのタッチターゲットが 44px 以上
- [ ] OS の「視差効果を減らす」(prefers-reduced-motion) でアニメ・浮き上がりが停止する
- [ ] OS の「透明度を下げる」(prefers-reduced-transparency) でブラーが切れ不透明面になる
- [ ] 主要テキスト・ボタンのコントラストが AA 以上（本文テキストが半透明で読みにくくない）

## 楽天API（キー設定時のみ・任意）

- [ ] `SERVER_RAKUTEN_APP_ID` を設定すると `公式API取得` ラベルの実データが表示される
- [ ] キー未設定・ネットワーク失敗時もアプリが落ちず、モック＋警告にフォールバックする
- [ ] ビルド成果物にキー文字列が含まれない（`grep -r "SERVER_RAKUTEN" dist/` で何も出ない）
