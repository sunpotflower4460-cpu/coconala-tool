# UX Principles

## Core UX

このアプリは、価格調査の複雑さを隠し、判断に必要なものだけを美しく見せる。

ユーザーは次の流れだけ分かればよい。

```text
商品名を入れる
↓
まとめて探す
↓
画像つきカードを見る
↓
比較に追加
↓
利益を見る
↓
元ページで確認
```

## Priorities

1. 見やすさ
2. 迷わなさ
3. とっつきやすさ
4. 価格カードの気持ちよさ
5. 利益判断の分かりやすさ
6. ココナラ販売時に伝わる見た目

## UI Rules

- 最初に見せるのはテーブルではなくカード。
- 詳細テーブルは比較画面で必要な人だけ見られるようにする。
- ボタンは大きく、ラベルは自然な日本語にする。
- 価格・画像・サイト名・元ページボタンはカード内で必ず見えるようにする。
- 価格が推定の場合は、小さくラベルを付ける。
- エラーは怖く出さず、代替アクションを見せる。

## Main Labels

- まとめて探す
- 比較に追加
- 元ページを見る
- 手動で追加
- 検索リンクを開く
- 利益見込み
- 要確認
- 狙い目
- 利益薄い

## Theme Direction

### Simple Pro

白基調。業務向け。物販ガチ勢に向く。

### Soft Market

やわらかい色。初心者向け。ココナラ映えしやすい。

### Dark Trader

黒基調。分析・輸出・トレーダー感。

### Natural Board

紙・ノート・木目風。やさしい研究ボード感。

## Mobile First

スマホでもカードが見やすいことを重視する。
PCでは比較ボードを広く使い、スマホではカード縦積みを基本にする。

## Good Empty State

初回画面では、何を入力すればよいかすぐ分かるようにする。

Example:

`例: SONY Walkman NW-A55 / Nintendo 3DS LL / JANコード / 商品URL`
