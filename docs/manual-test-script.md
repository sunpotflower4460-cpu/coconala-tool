# 手動テスト手順書

初めての方でもわかるよう、クリックごとに手順を記載しています。

---

## 準備

1. ターミナルを開いてください
2. アプリのフォルダに移動します

   ```
   cd coconala-tool
   ```

3. 依存パッケージをインストールします

   ```
   npm install
   ```

4. 開発サーバーを起動します

   ```
   npm run dev
   ```

5. ブラウザで `http://localhost:5173` を開いてください

---

## テスト 1: 検索してカードを表示する

1. 画面上部の入力欄に `PS5` と入力します
2. 「まとめて探す」ボタンをクリックします
3. **確認：** 商品カードが複数枚表示されることを確認します
4. **確認：** カードに「現在はサンプルデータです」というオレンジ色のバッジが表示されることを確認します
5. **確認：** 検索ショートカットカード（「メルカリで検索」「ヤフオクで検索」など）が表示されることを確認します

---

## テスト 2: カードを比較ボードに追加する

1. テスト 1 の続きから始めます（カードが表示された状態）
2. 任意のカードの「**比較に追加**」ボタンをクリックします
3. **確認：** 右側（スマホでは下）の「比較ボード」にそのカードが追加されます
4. さらに別のカードの「**比較に追加**」をクリックします
5. **確認：** 比較ボードに 2 枚になります
6. 比較ボードのカード右上の「×」をクリックします
7. **確認：** そのカードが比較ボードから消えます

---

## テスト 3: 利益を計算する

1. テスト 2 の続きから始めます（比較ボードにカードが 1 枚以上ある状態）
2. 比較ボードのカード上「**この価格を仕入れに使う**」をクリックします
3. **確認：** 右サイドの「利益計算」パネルの「仕入れ価格」が変わります
4. 「販売価格」の欄に手動で数字を入力します（例: `80000`）
5. **確認：** 利益金額・利益率が計算されて表示されます

---

## テスト 4: 手動でカードを追加する

1. 「**手動で追加**」ボタンをクリックします（カード一覧の右上にあります）
2. **URL 欄エラーテスト：**
   - URL 欄に `abc` と入力してフォーカスを外します
   - **確認：** 「URL形式が正しくありません」というエラーメッセージが出ます
3. **価格未入力テスト：**
   - URL 欄を空にしたまま他の欄を見ます
   - **確認：** 価格欄の下に「価格を入力しないと「価格不明」と表示されます」という案内が出ます
4. **重複 URL テスト（サンプルデータが表示されている場合）：**
   - URL 欄に `https://www.ebay.com/sch/i.html?_nkw=PS5+CFI-2000A01` を入力してフォーカスを外します
   - **確認：** 「同じURLのカードがすでに存在します」という警告が出ます
5. **正常追加テスト：**
   - URL 欄に `https://example.com/item1` を入力します
   - 価格欄に `¥55,000` と入力します
   - 「**比較に追加**」をクリックします
   - **確認：** カード一覧と比較ボードに新しいカードが追加されます
   - **確認：** 「手動追加」ラベルが付いています

---

## テスト 5: CSV をエクスポートする

1. 比較ボードにカードが 1 枚以上ある状態にします
2. 右サイドの「**CSVエクスポート**」セクションを見つけます
3. 「**比較中カードをCSV出力**」ボタンをクリックします
4. **確認：** CSV ファイルがダウンロードされます
5. ダウンロードしたファイルを Excel またはスプレッドシートで開きます
6. **確認：** 文字化けしていないことを確認します（BOM 付き UTF-8 形式）

---

## テスト 6: 履歴を保存・再開・削除する

1. 検索・比較を行った状態にします
2. 右サイドの「**リサーチ履歴**」セクションの入力欄に `PS5 テスト` と入力します
3. 「**保存**」ボタンをクリックします
4. **確認：** 「PS5 テスト」という名前で履歴に追加されます
5. ブラウザをリロード（F5）します
6. 「**再開**」をクリックします
7. **確認：** 保存時の状態（カード・比較ボード）が復元されます
8. 「**削除**」をクリックします
9. **確認：** その履歴が消えます

---

## テスト 7: テーマを切り替える

1. ヘッダー右上の「テーマ」セレクタを見つけます
2. 各テーマ（Simple Pro / Soft Market / Dark Trader / Natural Board）をクリックしてみます
3. **確認：** 画面の色が切り替わります
4. ブラウザをリロードします
5. **確認：** リロード後もテーマが維持されています

---

## テスト 8: 空状態の確認

1. ブラウザの LocalStorage をクリアするか、シークレットウィンドウを開きます
2. アプリを表示します（`http://localhost:5173`）
3. まだ「まとめて探す」を押していない状態を確認します
4. **確認：** 「商品名・型番・JAN・URLを入力して」という中央の案内が表示されます
5. 「まとめて探す」を押してから比較ボードを見ます
6. **確認：** 比較ボードが空のとき「まだ比較カードがありません」と表示されます
7. CSVエクスポートパネルを見ます
8. **確認：** 「比較ボードにカードを追加するとCSV出力できます」と表示されます

---

## テスト 9: モバイル表示の確認

1. ブラウザの開発者ツール（F12）を開きます
2. デバイスエミュレーション（スマートフォンアイコン）をクリックします
3. iPhone SE（375×667）相当のサイズを選択します
4. **確認：** 右サイドパネルが縦積みになる
5. **確認：** 長い商品名が折り返しまたは省略（`...`）されてレイアウトが崩れない
6. **確認：** ボタンが押せるサイズで表示される

---

## ビルド最終確認

1. ターミナルで以下を実行します

   ```
   npm run lint
   ```

   **確認：** エラーが出ない

2. 続けて以下を実行します

   ```
   npm run build
   ```

   **確認：** `dist/` フォルダが生成され、エラーが出ない
