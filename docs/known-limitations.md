# 既知の制限事項

- 既定では実マーケットプレイスのリアルタイムデータには未接続で、`サンプルデータ`（UI / 操作フロー確認用）と `楽天市場`（キー未設定時はモック＝公式API接続を想定した擬似データ）で動作します。
- 楽天市場 商品検索APIのみ、サーバー側（Cloudflare Pages Functions）に `SERVER_RAKUTEN_APP_ID` を設定すると `公式API取得` ラベルの実データに切り替わります。キー未設定・ネットワーク失敗・レート超過時は自動でモックにフォールバックします。
- APIキーはフロントエンド（`VITE_` バンドル）に置かず、サーバー側の環境変数でのみ管理します。ビルド成果物にキー文字列は含まれません。
- eBay / Yahoo!ショッピング / メルカリ・ヤフオクの実API接続は対象外です（メルカリ・ヤフオクは検索リンク生成と手動確認 / 手動追加を中心に扱います）。
- 検索表示由来の価格は `検索表示から推定` として扱い、確定価格ではありません。
- 手動追加 / CSV出力 / 履歴保存・再開は利用できます（ローカルブラウザ内保存）。
- 保存データは `localStorage` ベースのため、同じブラウザ内にのみ保持されます。
- CSVエクスポートはクライアントサイド処理であり、サーバー保存は行いません。
- 認証、課金、バックエンド永続化は対象外です。
- eBay / Yahoo!ショッピング等の追加公式API連携は次フェーズ（プレミアム）の別作業です。
- デプロイ先による対応範囲: Cloudflare Pages（推奨・楽天API正式対応）/ Vercel（静的UIのみ、実API連携には別途Function実装が必要）/ GitHub Pages（静的デモのみ）。詳細は `docs/deployment-guide.md` を参照してください。
